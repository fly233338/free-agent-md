## About the snapshots

Each entry identifies its upstream repository, SPDX license (or `NOASSERTION` in the machine-readable catalog), and a source link fixed to the observed commit. See [NOTICE.md](NOTICE.md) for attribution and takedown details.

The catalog is generated by software. Repository metadata is escaped, and instruction-file contents are never embedded here.

## Methodology

Candidates are collected from configurable GitHub star/activity tiers. Only the repository root and configured agent directories are inspected, without recursive tree scans. Heat combines repository-scale star percentile (50%), seven-day star-growth percentile (30%), and push recency with a 14-day half-life (20%). See [`config.yml`](config.yml) for the deterministic category rules and scan limits.

## Running locally

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e ".[dev]"
.\.venv\Scripts\pytest.exe
.\.venv\Scripts\free-agent-md.exe dry-run
```

Set `GITHUB_TOKEN` for authenticated API limits. Before publishing a fork, set `repository` in `config.yml`; GitHub Actions automatically uses `GITHUB_REPOSITORY`.
